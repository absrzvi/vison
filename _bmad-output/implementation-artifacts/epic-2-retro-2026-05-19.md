## What Went Well

- **DD-001 → implementation was straight-through.** Freya's design doc translated directly into stories with low rework. The design-first pattern is validated.
- **SSE over WebSocket held up.** Architecture pivot (REST + SSE replacing WS) survived first contact with live fleet feed in 2-1, then 8 more stories without protocol revision.
- **Playwright E2E (2-7) paid off immediately.** 2-8 and 2-9 inherited regression confidence for free.
- **Preferences panel + backend router (2-8) cleanly modeled in one story.** No scope creep; no follow-on patch story.

## What Hurt

- **2-2 scroll-to-top regression slipped to code review.** Filter tap behavior should have been verified in dev, not surfaced as a CR patch.
- **5 patches on 2-3 (fleet list sort).** Highest patch count of the epic. Likely either AC ambiguity or design-doc gap on sort tie-breakers / edge states. Worth a closer look in the next design-spec'd UI story.
- **Design ↔ dev sync gaps.** Spots where DD-001 didn't fully specify a state or interaction — interaction edges and empty/error states are the recurring miss.

## Key Insight

> The pattern **works**: design-first (DD-001) → small stories → Playwright E2E → SSE feed. The epic shipped 9 stories with one significant patch cluster (2-3). The improvement is not structural — it's **completeness in design specs around interaction edges** (sort tie-breakers, scroll restoration, empty/error states).
