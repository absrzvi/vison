Device Support

Yes — we support the Hailo-8

We have confirmed full support for the Hailo-8 M.2 accelerator on the Nomad Digital R5001C CCU. Our onboard inference stack is built on HailoRT 4.23 and TAPPAS 5.1.0, running inside a Docker container on the CCU's SYS2 compute partition. We reuse Hailo's open-source hailo-apps library for RTSP ingestion, YOLO detection, and multi-object tracking — which means the highest-risk custom engineering is already solved.

What the system does

6 AI capabilities, day one

The Passenger Intelligence Service runs continuously during each journey, producing structured, anonymised events that flow to staff interfaces and the cloud.

🧍
Live Occupancy
Real-time passenger headcount per coach. Colour-coded (green / amber / red) against configurable thresholds.
🧳
Luggage Detection
Item count per zone. Configurable timer triggers unattended bag alert when no owner detected nearby.
⬡
Congestion Mapping
Vestibule and aisle congestion scoring. Alerts when movement is obstructed — especially near doors.
🚪
Door Safety
Obstruction detection at all doors during boarding and departure. Direct alert to conductor with coach + door reference.
♿
Accessibility Detection
Wheelchair and pushchair detection. Automatic conductor alert + passenger portal update. Ramp coordination loop.
🔕
Maintenance Suppression
Alert suppression when in depot maintenance mode — prevents false positives during servicing.

System Architecture

5 containers · edge-first · privacy-by-design

All AI inference runs onboard the R5001C CCU (SYS2). Five Docker containers form a pipeline from raw camera streams to structured events. Raw video never leaves the train — only anonymised, timestamped events are synced to the cloud.

Physical
Inputs
VLAN 5
CCTV Cameras
Coach interior · vestibule · door cameras. RTSP streams per coach.
VLAN 7
TCMS (Stadler)
Train control alarms via SNMP. Journey ID · door state · speed · alerts.
VLAN 8
APC Counters
Automatic Passenger Counting — door-level boarding/alighting ground truth.
VLAN 2
ZFR (Door System)
Door open/close state. Corroborates vision-based obstruction detection.
RTSP streams + SNMP polls + door state
Onboard
Containers
R5001C · SYS2
Debian 12 · Docker
Container 1
rtsp-ingest
Receives RTSP streams from all CCTV cameras. Manages stream lifecycle and camera priority scheduling (P1 doors 10 fps · P2 interior 5 fps · P3 exterior 8 fps station-only). Passes decoded frames to inference.
Container 2
vlan-pollers
Polls TCMS via SNMP (alarms, door state, speed). Polls APC counters. Tracks journey ID — records journey_start_date when trip number first appears to prevent midnight-crossing bugs. Posts events to event-store.
Container 3 · Hailo-8 M.2
inference
Runs on the Hailo-8 accelerator (26 TOPS) via HailoRT 4.23 + TAPPAS 5.1.0.
Reuses hailo-apps (MIT) for YOLO detection, BYTETracker multi-object tracking, and pose estimation.
Outputs: headcount · luggage items · congestion score · door obstruction · wheelchair/pushchair detection · pose anomalies. Posts structured detection events to event-store.
Container 4
fusion
Merges vision detections with APC ground truth and TCMS alarm state.
Runs the suppression state machine — suppresses alerts during maintenance mode, depot coupling, GPS-invalid windows.
Applies configurable alert thresholds (dwell timer for unattended bag, congestion score threshold, occupancy %).
Posts final fused events to event-store.
Container 5 · Single writer · REST + WebSocket API
event-store
The only container with an HTTP port. All other containers write events via POST /api/v1/events — never direct DB access.
Persists to SQLite WAL (journey-scoped). Serves REST + WebSocket to all onboard interfaces (VLAN 30 staff · VLAN 10 passenger portal).
Maintains a sync_cursor — batches confirmed events for cloud sync via SYS1, then truncates safely. Keeps last 3 journeys as debug buffer.
REST · WebSocket · onboard VLAN 30 (staff) / VLAN 10 (passenger)
